Architecture

The One-Engine Law

Write the rule once. Run it everywhere. Prove it mathematically.

Write Once, Prove Everywhere

Every validation rule in the CanonFlow ecosystem is written exactly once in pure functional F#. This single source of truth then compiles to two identical execution targets — with zero semantic drift between them.

  • 1 Write the rule once in pure functional F#
  • 2 Run natively on .NET 10 servers via AOT compilation
  • 3 Compile to WebAssembly via Fable for offline browser validation
  • 4 Zero drift between server and client — same logic, same results

F# — Quick Start 
// 1. Define a rule as a pure function
let validatePlaceOfSupply (invoice: GSTInvoice) : VerdictEnvelope =
    match invoice.SupplierState = invoice.RecipientState with
    | true  → VerdictEnvelope.Pass "Intra-state: CGST+SGST applies"
    | false → VerdictEnvelope.Pass "Inter-state: IGST applies"

// 2. Run on server (AOT) or browser (WASM) — same result
let result = validatePlaceOfSupply sampleInvoice
// VerdictEnvelope { Status = Pass; Message = "Intra-state: CGST+SGST applies" }

How to Contribute

1. Fork & Clone — Pick a project from the Foundation org.
2. Find an Issue — Look for good-first-issue or help-wanted labels.
3. Write F# — Pure functions, discriminated unions, pattern matching.
4. Test — All rules must have snapshot + property-based tests.
5. Submit PR — Include ADR reference if architectural.
